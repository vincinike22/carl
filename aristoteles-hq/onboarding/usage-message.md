# How to Use Aristoteles HQ

## Mission-control
Use for:
- assigning work
- requesting synthesis
- coordinating across domains
- checking status
- routing open questions

## Decisions-for-aurelius
Use for:
- final decision briefs
- recommendation + rationale + tradeoffs
- requests for explicit approval

Format:
- Decision needed
- Recommended option
- Why this option
- Main risks
- What happens next after approval

## Daily-briefing
Use for one compact update containing:
- what moved today
- top open risks
- tomorrow's priorities
- decisions pending from Aurelius

## Specialist channels
Use each specialist room for domain work only.

- `leonardo-studio` → concepts, positioning, messaging, naming, creative options
- `michelangelo-workshop` → deliverables, execution plans, build steps
- `isaac-lab` → research, evidence, validation, comparisons
- `carl-red-team` → objections, risk, blind spots, failure modes

## Message standard
Every substantial message should ideally contain one or more of:
- evidence
- recommendation
- risk
- decision request
- next action

Avoid vague discussion with no operational output.
