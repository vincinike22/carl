# Aristoteles HQ

Lean multi-agent Discord operating system for Aurelius.

## Purpose
Aristoteles HQ is a practical decision-and-execution structure:
- Aurelius is the founder and final decision-maker
- Aristoteles is the strategic synthesis and coordination layer
- Specialist agents operate in clear domains with low noise

## Included
- Discord server blueprint
- Roles and permission model
- Channel structure
- Onboarding text
- Agent prompts
- Operating cadence
- Exact next-step setup instructions

## Core design principles
- Low noise over constant chatter
- Clear ownership per channel
- Aristoteles summarizes, specialists feed inputs
- Decision traffic gets compressed before it reaches Aurelius
- Working v1 over overbuilt theater

## Folder map
- `discord/` — roles, channels, permissions, setup sequence
- `onboarding/` — welcome and usage copy
- `agents/` — prompt/config text for each agent
- `ops/` — operating model, cadences, briefing formats

## Recommended first move
1. Create roles and channels from `discord/server-blueprint.md`
2. Post onboarding copy from `onboarding/`
3. Assign agent prompts from `agents/`
4. Start with Aristoteles + Isaac + Carl + one builder lane
